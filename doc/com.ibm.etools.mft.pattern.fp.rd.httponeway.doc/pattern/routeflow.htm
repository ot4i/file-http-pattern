Route subflow for the Record Distribution to HTTP: one-way pattern

Details about the Route subflow for the Record Distribution to HTTP: one-way pattern.

The behavior of the Route subflow depends on the value of the Routing pattern parameter:

  • Single Destination

    If you select Single Destination, then a single HTTPRequest node is created and configured with the values of the Web Service URL pattern parameter. The routing flow is a subflow that includes a Passthrough node. Both default and route terminals are included so that this subflow is consistent with all other types of routing subflow, but only the single route terminal is used.

  • Specify Routes

    If you select Specify Routes, then you must define at least one route.

    To define a route, assign values to the following group of three parameters:

    • Key Location: An XPath expression that defines a key location within the current message element. For more information about the type of expression to use, see "Route node" and "Using XPath" in the IBM Integration Bus documentation.
    • Key Value: A value that must match the key location if this route is taken.
    • Web Service URL: The URL of an HTTP web service that can process the message.

    For each route that is defined, an Output terminal for the Route subflow and an HTTPRequest node are created and connected.

    The Route subflow contains a Route node that is configured with a filter that routes to the first Output terminal for which the condition Key Location = Key Value is true. If a message element does not match any of the routes, this situation is recorded in a variable in the integration node environment tree, and the message is passed to the Default terminal.

    If the XPath expression that defines a Key Location includes a namespace qualifier, the tag that defines the namespace must be added to the Namespace table.

    You might see a warning on the Route node that is similar to the following message:

    Filter table: Filter pattern: The element_name schema element was not found in the Key Location = Key Value XPath.

    This warning does not cause any problems when you deploy and run your pattern instance.
