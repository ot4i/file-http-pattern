Parameters for the HTTP one-way pattern

Sample
Pattern Parameter Default Description
Include Sample None

Specified the sample type if required

Options:

  • None: Indicates that a sample is not required
  • Single Destination: Includes a sample with no routing capabilities
  • Specific routing: Includes a sample with specific routes
Input file
Pattern Parameter Default Description
Input Directory

The location of the input file.
Filename pattern *

A file name or string containing optional wildcard characters (asterisk (*) or question mark (?)) identifying the file or files to process from the input directory.

Use FTP Cleared

This parameter determines whether FTP is used to retrieve remote files.

FTP configurable service

This configurable service defines how FTP is used to retrieve a remote file.

Routing
Pattern Parameter Default Description
Routing Single Destination

This parameter defines the routing method to use. Valid values are:

  • No routing
  • Specify routes
Single Destination
Pattern Parameter Default Description
Web Service URL

This parameter defines the HTTP web service to which all output messages are sent.

Specify Routes
Pattern Parameter Default Description
Default Web Service URL

This parameter defines the HTTP web service to which messages are sent when the criteria for the Specify routes property value are not met.

Namespace Table

Namespace table define the namespace prefixes and the namespace URIs that you can use in the XPath expressions that define the key locations for routes. If a row is included, both of the parameters must be included.

You can add, edit, and delete values, and change the order of the values in the table.
Routing Table

Defines the routing XPATH expressions' key, value and HTTP web service URL

All the pattern parameters are listed with their default values. Where no default is indicated, the default value is an empty string.
