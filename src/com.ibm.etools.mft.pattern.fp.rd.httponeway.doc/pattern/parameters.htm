Parameters for the Record Distribution to HTTP: one-way pattern

Input file
Pattern parameter M C Default Description
Input directory Yes Yes  

The location of the input files.
Filename pattern Yes Yes *

A file name or string that contains optional wildcard characters (asterisk (*) or question mark (?)), that identify the file or files to process from the input directory.
Use FTP Yes Yes Not selected

If the check box is selected, FTP is used to retrieve remote files. If the check box is not selected, the files are accessed directly from the input directory.
FTP configurable service No Yes ftpservice

The name of the configurable service that defines how FTP is used to retrieve a remote file.
Routing
Pattern parameter M C Default Description
Routing No No Single Destination

The routing method to use. The following are valid values:

  • Single Destination
  • Specify Routes
Single Destination
Pattern parameter M C Default Description
Web Service URL No Yes

The HTTP web service to which all output messages are sent when Single Destination is selected as the routing method.
Specify Routes
Pattern parameter M C Default Description
Default Web Service URL No Yes

The HTTP web service to which messages are sent when the criteria for the Specify Routes property value are not met.

The parameters in the following Namespace table define the namespace prefixes and the namespace URIs that you can use in the XPath expressions that define the key locations for routes. If a row is included, both of the parameters must be included.

You can add, edit, and delete values, and change the order of the values in the table.

Namespace table parameter M C Default Description
Namespace prefix Yes No

The tag for a namespace.
Namespace URI Yes No

The namespace for the associated tag.

The Namespace table defines the namespaces that are available for you to use in XPath expressions in the Routing table.

Routing table parameter M C Default Description
Key location Yes No

An XPath expression that defines a location within the record that is being propagated. For example: $Root/XMLNSC/submitPORequest/partNo

For more information about the type of expression to use, see "Route node" and "Using XPath" in the IBM Integration Bus documentation.

Namespaces can be included if they are specified in the optional Namespace table.

Key value Yes No

The Key location field in the incoming message must match this value in order to use this route.
Web Service URL Yes Yes

The target HTTP web service URL for an incoming message that matches the associated key value.

All of the pattern parameters are listed with their default values.

The column headed M indicates whether the property is mandatory. The column headed C indicates whether the property is configurable, meaning that you can change the value when you add the message flow to the broker archive (BAR) file.

Where no default is indicated the default value is an empty string.
