Tasks to complete after generating the Record Distribution to HTTP: one-way pattern

Complete the following tasks after you have generated the pattern:

  1. Create the broker archive (BAR) file. Add the generated message flow and subflows to your BAR file. In the BAR file editor, you must select the Compile and in-line resources option before you build the BAR file. If you have used a message model to define how records are parsed, you must add the message model to the BAR file. For more information, search for "Creating a BAR file" in the IBM Integration Bus documentation.
  2. Configure the BAR file. Review the default options and configure any changes required for your environment.
  3. If you have chosen to use FTP to access the input files, you must create an FtpServer configurable service on the target integration node so that the FileInput node can connect to the remote FTP server. The name of the configurable service must match the name that you specified in your pattern configuration. For more information, search for "FtpServer configurable service" in the IBM Integration Bus documentation.
  4. Deploy the BAR file. For more information, search for "Deploying a BAR file" in the IBM Integration Bus documentation.
