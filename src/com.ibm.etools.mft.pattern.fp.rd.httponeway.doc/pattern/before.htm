Tasks to complete before you generate the Record Distribution to HTTP: one-way pattern

Before you generate a pattern instance, you must complete the following tasks:

  • Confirm that this pattern meets the business requirements and can be supported by both the system that creates the files and by the destination HTTP web services.
  • Identify the purpose of the pattern application and give the pattern instance an appropriate name.
  • Decide how records are detected and, if required, prepare a message model to define how records are parsed. For more information, search for "Recognizing file records as messages to be parsed" in the IBM Integration Bus documentation.
  • Decide whether validation is required and, if required, prepare a message set to define how records are parsed. For more information, search for "Validating messages" in the IBM Integration Bus documentation.
  • Determine whether the files that are to be processed are local or remote. If remote files are to be processed, identify the name of the FtpServer configurable service. For more information, search for "FtpServer configurable service" in the IBM Integration Bus documentation.
  • Determine the number of possible destinations for a record:
    • If there is only one destination, set the Routing parameter to Single Destination and supply the HTTP web service URL of the destination.
    • If there is more than one destination, set the Routing parameter to Specify Routes and define the routing rules and destinations.
