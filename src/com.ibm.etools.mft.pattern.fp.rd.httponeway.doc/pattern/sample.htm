Using the sample data with the Record Distribution to HTTP: one-way pattern

If you want to test the pattern without creating your own input data and HTTP web service, you can use the sample data that is provided with the pattern. When you select a sample configuration, the required parameters are automatically completed with values that match the sample data.

Note: To run the HTTP web services that are provided with the pattern, you must have the Node.js runtime environment installed on your computer. The Node.js runtime environment can be downloaded from the Node.js web site: http://nodejs.org/download/.

There are two sample configurations available with the Record Distribution to HTTP: one-way pattern.

If you select Single Destination as the sample configuration, then the records that are extracted from the files are all sent to the Default web service.

If you select Specific routing as the sample configuration, then the records that are extracted from the files are examined before they are routed to a web service:

  • If the Country field in the record is set to "United Kingdom", the record is routed to the UK web service.
  • If the Country field in the record is set to "United States", the record is routed to the US web service.
  • If the Country field in the record is not set to "United Kingdom" or "United States", the record is routed to the Default web service.
You can extend this sample configuration by adding additional routing configurations.

The following sample resources are created when you generate the application from the pattern:

  • An input directory where you add the files to be processed by the application:
    • On Windows: temp_dir\file_to_http_sample
    • On Linux: temp_dir/file_to_http_sample
    where temp_dir is your operating system's temporary directory.

  • Three sample files that contain user records:
    • One record has a Country field set to "United Kingdom".
    • One record has a Country field set to "United States".
    • One record has a Country field set to "Canada".

  • A Node.js web server with three web services available from the following URLs:
    • UK web service: http://localhost:port/country/uk.html
    • US web service: http://localhost:port/country/us.html
    • Default web service: http://localhost:port/country/default.html
    • where port is an available port. By default the port will be 3000, unless this port is not available.
